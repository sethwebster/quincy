# Quincy — Agent Workspace

Quincy is a cross-platform markdown editor built for focus, beauty, and speed. It runs as a native desktop app via Electrobun and syncs documents in real-time using Convex. The design language is glassy, modern, and intentional — every detail earns its place.

## Product Vision

Three editing modes, one experience:

1. **Rich Text** — Markdown rendered, syntax invisible. Users write without knowing it's markdown.
2. **Side-by-Side** — MD source on the left, live rendered preview on the right.
3. **Source + Helpers** — Raw markdown with syntax hints, toolbar assists, and contextual suggestions.

The editor must be delightful. Fast. Beautiful. Thoughtful small touches everywhere — but always in service of the writing experience, never distracting from it.

## Tech Stack

| Layer | Technology |
|---|---|
| Desktop runtime | [Electrobun](https://electrobun.dev) |
| UI framework | React (within Electrobun's browser context) |
| Styling | Tailwind CSS v4 |
| Animation | Framer Motion |
| Real-time sync | Convex |
| Language | TypeScript |

### Design Language
- **Glassy** — frosted glass panels, blur effects, translucent surfaces
- **Light-aware** — surfaces respond to ambient light metaphors
- **Typographic** — a writing tool must have impeccable type
- **Motion** — transitions feel physical, not mechanical (Framer Motion)
- **Minimal chrome** — the document is the UI

## Critical Notes

### Electrobun Architecture
- **Main process** runs in Bun — no DOM, handles native OS APIs and window management
- **Renderer process** runs in the system WebView (or CEF on Linux) — this is where React lives
- **RPC** between main ↔ renderer is typed via Electrobun's RPC system — use it for all cross-process calls
- Never access DOM APIs from main process; never access Bun-only APIs from renderer

### Convex Sync
- Convex handles all document persistence and real-time sync
- Use `useQuery` / `useMutation` from `convex/react` in renderer
- Documents have optimistic updates — keep local state minimal
- Auth via **Convex Auth** with magic link (email OTP) — no passwords, no OAuth

### Editor Mode Transitions
1. Mode changes must preserve cursor position and selection
2. No content loss on any mode switch — ever
3. Transition animated with Framer Motion (not instant snap)
4. Mode state lives in app-level context, not individual editor components

### Performance Rules
- The editor must feel instantaneous — 0 perceptible lag on keypress
- Rendering the preview pane debounces at 100ms max
- Heavy markdown parsing runs in a Web Worker
- No re-renders of the full editor tree on every keystroke (use fine-grained subscriptions or `useSyncExternalStore`)

## Architecture Decisions

### Editor Core
- Rich text mode: [TipTap](https://tiptap.dev) (ProseMirror-based, excellent markdown support)
- Source/split mode: [CodeMirror 6](https://codemirror.net/6/) (fast, extensible, great for MD syntax)
- Markdown parsing/rendering: [unified](https://unifiedjs.com) / remark + rehype pipeline

### State
- Document content: Convex (source of truth for persistence)
- Editor local state (cursor, selection, mode): React context + refs
- UI state (sidebar open, theme, preferences): Electrobun main process (persisted to disk)

### Styling
- Tailwind v4 with CSS custom properties for the glass design system
- `backdrop-filter: blur()` for glass surfaces — test performance on all targets
- Dark mode first; light mode adaptive
